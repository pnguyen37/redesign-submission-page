Responsive Redesign

Overview

This is a redesign project in which I attempted to improve the usability and accessibility of an existing interface. I started by sketching and wireframing to address the major usability issues in the layout. Then, I made a visual style guide and high fidelity mockup to finalize the style of the new design before finally implementing the page.

Original Page

The interface I chose is the ordering page of the restaurant East Side Pockets. I chose this website because the interactive elements were not intuitive to use in comparison to other restaurant ordering pages, and there were alignment and spacing issues that I felt could be addressed in a more responsive design. Identifying Usability Problems

Usability Issues

  • Poor affordances / learnability: Some buttons in the banner don’t seem clickable and just look like images, so banner might get ignored completely. Order receipt looks like it could be a button but is not. At a glance, menu seems more like plain text than an interactive page.

  • Not fully responsive: The nav bar disappears for small screens and some of the text in the nav bar gets pushed off as the screen gets smaller. The order function doesn't show up on smaller screens if you don't already have something in the cart.

  • Alignment / spacing: Menu images interfere with left alignment of text. Some images are big enough to take up the whole screen. There is a lot of unused space on right side when the order is empty.

  • Inefficiency: When the order receipt gets too large, the button to order and some of the food items get cut off and you can't scroll the receipt without scrolling to the top or bottom of the page. Images need to be clicked on separately from the menu item to be seen more clearly. You can't navigate between sections without going to the top first.

  • Memorability: Too many redirection links in the banner and nav bar may make it difficult for the user to remember where they should go or which link they should use to get there. Inconsistency with hover effects may make the user forget what is clickable, especially for different links in the footer.

Accessibility Issues

    After analyzing the page using WebAIM WAVE, I found that the page is not very accessible to users with vision impairments. I agree with the problems identified because most of the issues are objective issues that would cause problems for assitive technologies. Here are some of the issues I found:

  • The text contrast is too low because the shade of green is too light for a white background.

  • The language of the document is not identified so screen readers can't read the page in the correct language.

  • There are no aria labels, and some images are missing alternative text, so images and icons wouldn’t be read out loud.

  • There is no heading structure, which creates navigation issues for assitive technologies.

  • Some of the dropdown menus require a click and aren’t accessible through the keyboard, which creates problems for those with mobility impairments.

Visual Redesign

Sketches

I made 9 sketches (left image) to brainstorm new designs for the page. Afterwards, I combined the different ideas into one final sketch (right image), which had some of my favorite components, like a banner with dropdown buttons, a two-column display, clickable cards with images, vertical scrolling, and a footer with navigation links. I drew the sketches in a vertical orientation to fit the full screen.

Low-Fidelity Wireframes

I used my final sketch as the foundation for my wireframes, which demonstrate the new layout for Desktop View (top page), Tablet View (middle page), and Phone View (bottom page). I addressed the usability issues in my annotations, which can be read below. Some of the key fixes were addressing accessibility issues like the lack of heading structure, and usability issues like poor affordances and misalignment. I addressed the poor affordances issue by making interactive components more obvious to the user, and spacing issues by reorganizing the placement of content.

Visual Style Guide

I made a style guide that I thought would fit the restaurant's brand, and chose to stick to similar colors as the original website to stay on theme. I chose a simpler font so that it would not be too distracting, especially since I chose some bright colors. I also made sure to include hoverable components to ensure consistency in the hover effects.

Takeaways

After going through this redesign process, I learned that there are a lot of factors to consider when choosing a design. For example, I had to think about components that would be accessible, have good affordances, follow convention, and also have a consistent and clean style. Convention in particular was tricky because when I was brainstorming design ideas, it was hard to come up with original ways to display menus when the two-column format is so widely recognized. I also learned how important testing is, especially when testing for accessibility, because to improve the page I needed to consider different users and how they might interact with the page. Overall, it was a good learning experience and I liked the iterative nature of the design process.
